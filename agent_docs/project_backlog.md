- `M6-001` - Rewrite `parse_backlog` around K9 canonical H3 delimiters with structured done-detection.
  Status: completed
  Scope: `src/commands/bootstrap_k9.rs`, `tests/bootstrap_k9.rs`, new fixture files derived from real Free-AI-SSD K9 entries (`### C7`, `### C27`, `### C6`).
  Notes: Covers Bugs 1, 2, and 3 from the parser analysis report together — they are entangled once H3 is the unit of iteration. K9 canonical (`K9-Claude-Framework/docs/file-structure.md:156-167`) uses `### Title` as the item delimiter with bolded-field bullets in the body, including `- **Status.** <triaged | planning | in-progress | blocked | done>`. That is the only format the parser must support. Free-AI-SSD's backlog correctly follows this convention. Current parser splits on top-level `- ` bullets and only catches the rarely-used trailing `Status: done` line, which produced 519 tasks from a 49-task Free-AI-SSD backlog and zero done-skips. Fix: iterate on `### Title` headings, accumulate everything between headings as body, detect done in priority order (heading suffix `— **done**`, strikethrough `~~...~~` heading, body `**done** PR #N`, bulleted `- **Status.** done`, plus the existing trailing `Status: done` line as a tolerated fallback). K9 status vocabulary maps to memhub's `tasks.status` as `done → completed`, `blocked → blocked`, everything else (`triaged | planning | in-progress`) → `open`. Regression assertion: Free-AI-SSD-style backlog yields ~50 tasks, not ~500. Bug 5 (decisions H2 discriminator) and Bug 6 (UTF-8 mojibake) from the same report are intentionally deferred. Gates M6-003. memhub's own `agent_docs/project_backlog.md` does not currently follow K9 canonical; that mismatch is M6-004's scope, not this task. Landed: `parse_backlog` and `classify_backlog_body` replaced with H3-driven parser plus `parse_backlog_heading` / `classify_backlog_task` / `line_has_inline_done_pr_marker` / `extract_bulleted_status` / `extract_legacy_status` / `map_k9_status` / `map_legacy_status` helpers in `src/commands/bootstrap_k9.rs`. 9 K9-canonical unit tests replaced 2 stale bullet-format ones (canonical parse, all four done-detection patterns, vocab mapping, legacy `Status:` fallback, preamble-bullet regression, no-status default, synthetic Free-AI-SSD shape regression). `SAMPLE_BACKLOG` in `tests/bootstrap_k9.rs` migrated to K9 canonical preserving its assertion counts. 156 tests pass; `cargo build` clean. - `M6-002` - Accept em-dash in `strip_date_prefix` and extract decision date into `decided_at`.
  Status: completed

- `M6-004` - Migrate memhub's own `agent_docs/project_backlog.md` and `project_decisions.md` to K9 canonical structural delimiters.
  Status: open
  Scope: `agent_docs/project_backlog.md`, `agent_docs/project_decisions.md`.
  Notes: memhub currently dogfoods a divergent format: top-level `- ` bullets with ASCII-hyphen titles in the backlog, and `## YYYY-MM-DD - Title` with ASCII hyphen in the decisions. K9 canonical (`K9-Claude-Framework/docs/file-structure.md:156-208`) specifies `### Title` H3 delimiters for backlog items and `## YYYY-MM-DD — Title` with em-dash for decisions. Closing this gap matters because memhub claims to integrate with K9 yet ships a non-canonical example of K9 usage in its own continuity files. Scope is structural alignment only: (a) rewrite backlog item delimiters from `- ID - title` to `### ID — title`; (b) swap decision heading separators from ASCII hyphen to em-dash; (c) preserve each existing item's prose `Notes:` paragraph below the new heading rather than decomposing it into K9's bulleted `- **Scope.** ... - **Status.** ...` slots. Mechanical conversion preserves all current information; stylistic restructuring of `Notes:` into K9's bulleted slot vocabulary is explicitly deferred — the dense narrative is more useful for memhub's milestone-history use case than bulleted slots would be. Independent of M6-001/M6-002/M6-003; can land in any order. Once it lands, running `bootstrap-k9` on memhub itself becomes a valid second smoke-test target.
